Push all work to Git.

1. Run `git status` and `git diff` to understand what has changed.
2. Determine whether the changes fall into multiple distinct areas (for example: different features, bug fixes, configuration changes, documentation updates, or other unrelated work).
3. If there are multiple distinct areas of change:
   - Group related files and changes together.
   - Create separate commits for each group.
   - Use clear, descriptive commit messages for each commit.
4. If all changes are related, create a single commit with an appropriate commit message.
5. Stage and commit each group individually.
6. Push all commits to the remote repository.

Always push to the remote at the end.
